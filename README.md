# TechRobot — Информационный сайт о технологиях и робототехнике

> Стек: HTML5 · CSS3 · JavaScript ES6+ · Node.js · Express · MySQL · JWT  
> Без сторонних фронтенд-фреймворков

---

## Описание проекта

**TechRobot** — многостраничный информационный портал о современных технологиях и робототехнике. Сайт охватывает шесть ключевых направлений: искусственный интеллект, робототехника, интернет вещей, VR/AR, биотехнологии и беспилотные системы.

### Основные возможности

| Раздел | Функции |
|--------|---------|
| **Главная** | Hero с анимацией счётчиков, категории, последние статьи и новости, превью каталога роботов, CTA регистрации |
| **Статьи** | Сетка из 16 статей, фильтр по 6 категориям, поиск, сортировка по дате/популярности/длине |
| **Статья** | Полный HTML-контент, мета-данные, теги, кнопка «В избранное», блок похожих статей |
| **Новости** | Лента из 25 новостей с датами, тегами, разворачиваемым полным текстом, фильтр по категориям |
| **Каталог роботов** | 12 роботов, фильтр по 6 типам, поиск, модалка с характеристиками |
| **Видеотека** | 8 RuTube iframe-плееров, фильтр по категориям |
| **Блог сообщества** | Авторские статьи пользователей с премодерацией. Создание/редактирование в WYSIWYG-редакторе с превью, фильтр по 6 категориям, пагинация. HTML-контент санитизируется через `sanitize-html` (сервер) + DOMPurify (клиент) |
| **Админ-панель** | Очередь модерации блога: одобрение или отклонение статей с указанием причины. Доступна только пользователям с `is_admin = 1` |
| **Поиск** | Живой поиск по статьям/новостям/роботам/блогу с подсветкой совпадений |
| **Личный кабинет** | Профиль, избранное, история просмотров, мои статьи блога, редактирование данных |
| **О нас / Контакты** | О проекте, автор, контакты (Telegram, ВКонтакте, Discord, Email), форма обратной связи |

## Архитектура

```
Браузер  ──→  Nginx (порт 80)
                ├── /           → статика (HTML/CSS/JS/JSON/изображения)
                └── /api/*      → Node.js Express (порт 3000)
                                    └── MySQL (база techrobot)
```

### Авторизация

- **Регистрация и вход** — через REST API (`POST /api/register`, `POST /api/login`)
- **Пароли** — хешируются на сервере через **bcrypt** (10 раундов)
- **Сессия** — **JWT-токен** (срок действия 7 дней), хранится в `localStorage`
- **Избранное и история** — сохраняются в MySQL, синхронизируются с сервером
- **Оптимистичные обновления** — UI обновляется мгновенно, данные синхронизируются с сервером в фоне
- **Кросс-устройственная синхронизация** — `GET /api/me` подтягивает свежие данные при открытии профиля

### API-эндпоинты

Условные обозначения: ✅ — нужен JWT, 👑 — нужен JWT + флаг `is_admin`.

**Авторизация и профиль**

| Метод | Путь | Auth | Описание |
|-------|------|------|----------|
| `POST` | `/api/register` | — | Регистрация нового пользователя |
| `POST` | `/api/login` | — | Вход в аккаунт |
| `GET` | `/api/me` | ✅ | Получить данные текущего пользователя (включая `isAdmin`) |
| `PUT` | `/api/profile` | ✅ | Обновить профиль (имя, био, аватар) |
| `POST` | `/api/favorites/:id` | ✅ | Добавить/убрать статью из избранного |
| `POST` | `/api/history/:id` | ✅ | Записать просмотр статьи |
| `DELETE` | `/api/history` | ✅ | Очистить историю просмотров |
| `POST` | `/api/contact` | — | Отправить форму обратной связи |

**Блог сообщества**

| Метод | Путь | Auth | Описание |
|-------|------|------|----------|
| `GET` | `/api/blog` | — | Список одобренных статей (`?page=&limit=&category=`) |
| `GET` | `/api/blog/:id` | опц. | Одна статья (approved публично, pending/rejected — только автор/админ). Инкремент `views` для approved |
| `GET` | `/api/blog/my` | ✅ | Свои статьи всех статусов (для личного кабинета) |
| `POST` | `/api/blog` | ✅ + лимит 5/час | Создать черновик. Контент санитизируется через `sanitize-html` |
| `PUT` | `/api/blog/:id` | ✅ автор | Редактирование своей `pending`/`rejected` статьи; после — статус сбрасывается в `pending` |
| `DELETE` | `/api/blog/:id` | ✅ автор / 👑 | Удалить статью |
| `GET` | `/api/admin/blog/pending` | 👑 | Очередь модерации (старые первыми) |
| `POST` | `/api/admin/blog/:id/review` | 👑 | Одобрить/отклонить: `{action:'approve'\|'reject', reason?}` |

## База данных (MySQL)

### Таблица `users`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT AUTO_INCREMENT | Первичный ключ |
| `name` | VARCHAR(100) | Имя пользователя |
| `email` | VARCHAR(255) UNIQUE | Email (логин) |
| `password_hash` | VARCHAR(255) | Хеш пароля (bcrypt) |
| `avatar` | VARCHAR(500) | URL аватара |
| `bio` | TEXT | Описание профиля |
| `is_admin` | TINYINT(1) DEFAULT 0 | Флаг администратора (доступ к `admin.html`) |
| `created_at` | TIMESTAMP | Дата регистрации |

### Таблица `favorites`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT AUTO_INCREMENT | Первичный ключ |
| `user_id` | INT FK → users.id (CASCADE) | Пользователь |
| `article_id` | INT | ID статьи |
| `created_at` | TIMESTAMP | Дата добавления |

UNIQUE KEY `(user_id, article_id)` — статью нельзя добавить дважды.

### Таблица `view_history`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT AUTO_INCREMENT | Первичный ключ |
| `user_id` | INT FK → users.id (CASCADE) | Пользователь |
| `article_id` | INT | ID статьи |
| `viewed_at` | TIMESTAMP | Дата просмотра |

### Таблица `contact_messages`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT AUTO_INCREMENT | Первичный ключ |
| `name` | VARCHAR(100) | Имя отправителя |
| `email` | VARCHAR(255) | Email для ответа |
| `subject` | VARCHAR(50) | Тема (`question` / `suggestion` / `error` / `cooperation` / `other`) |
| `message` | TEXT | Текст сообщения (10–1000 символов) |
| `created_at` | TIMESTAMP | Дата отправки |

### Таблица `blog_posts`

Пользовательские статьи с премодерацией.

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT AUTO_INCREMENT | Первичный ключ |
| `user_id` | INT FK → users.id (CASCADE) | Автор |
| `title` | VARCHAR(200) | Заголовок (5–200 символов) |
| `cover_url` | VARCHAR(500) | URL обложки (опционально, только https://) |
| `category` | VARCHAR(50) | Категория: `ai`, `robotics`, `iot`, `vrar`, `biotech`, `drones` |
| `excerpt` | VARCHAR(500) | Краткое описание (20–500 символов) |
| `content` | MEDIUMTEXT | HTML-контент (100–50000 символов, очищается через `sanitize-html`) |
| `status` | ENUM | `pending` (на модерации) / `approved` (опубликована) / `rejected` (отклонена) |
| `rejection_reason` | VARCHAR(500) | Причина отклонения (заполняется при `status='rejected'`) |
| `created_at` | TIMESTAMP | Дата создания |
| `reviewed_at` | TIMESTAMP NULL | Дата решения модератора |
| `reviewed_by` | INT FK → users.id (SET NULL) | Кто проверял (админ) |
| `views` | INT DEFAULT 0 | Счётчик просмотров (инкремент только для approved) |

Индексы: `idx_status`, `idx_user_status (user_id, status)`, `idx_created (created_at DESC)`.

**Жизненный цикл статьи:**
1. Пользователь создаёт → `pending`
2. Админ одобряет → `approved` (видна всем, инкремент `views`)
3. ИЛИ админ отклоняет с причиной → `rejected` (видна только автору и админу, с причиной в баннере)
4. Автор может редактировать `pending`/`rejected` → статус автоматически сбрасывается в `pending` для повторной модерации

## Установка и запуск

### Развёртывание на VPS (Ubuntu)

```bash
# 1. Клонировать репозиторий
git clone https://github.com/StepanOlohov/techrobot.git /var/www/html

# 2. Запустить автоматическую установку
cd /var/www/html/server
bash setup.sh
```

Скрипт `setup.sh` автоматически установит Nginx, Node.js 20, MySQL, PM2, создаст базу данных, запустит API-сервер и настроит Nginx как обратный прокси.

### Локальная разработка

```bash
# 1. Запустить MySQL и создать базу данных techrobot

# 2. Установить зависимости и запустить API
cd server
npm install
node server.js

# 3. Запустить статику (в другом терминале)
cd ..
python -m http.server 8080

# Откройте: http://localhost:8080
```

> При локальной разработке API доступен на `http://localhost:3000`, статика на `http://localhost:8080`. Для работы без Nginx-прокси нужно изменить `API_BASE` в `js/auth.js`.

### Переменные окружения

Сервер читает следующие переменные (есть дефолты в коде, но для production обязательно задавать через `.env` или systemd):

| Переменная | По умолчанию | Назначение |
|------------|--------------|------------|
| `PORT` | `3000` | Порт API-сервера |
| `JWT_SECRET` | `TechRobot_JWT_Secret_2024!` | Секрет для подписи JWT |
| `DB_HOST` | `localhost` | Хост MySQL |
| `DB_USER` | `techrobot` | Пользователь MySQL |
| `DB_PASSWORD` | `TechRobot2024!` | Пароль MySQL |
| `DB_NAME` | `techrobot` | Имя базы |

## Администрирование

### Назначение администратора

Чтобы пользователь увидел пункт «🛡 Админка» в шапке и получил доступ к `admin.html`, нужно поставить флаг `is_admin = 1` в БД.

**Вариант 1 — через MySQL CLI:**

```bash
mysql -u techrobot -p techrobot
```

```sql
UPDATE users SET is_admin = 1 WHERE email = 'admin@example.com';
```

**Вариант 2 — одной строкой:**

```bash
mysql -u techrobot -p'TechRobot2024!' techrobot -e \
  "UPDATE users SET is_admin = 1 WHERE email = 'admin@example.com';"
```

После этого пользователь должен **перелогиниться** (или подождать, пока сработает `AuthModule.syncCurrentUser()` при следующем открытии профиля или админ-панели).

### Применение миграции вручную

Изменения схемы БД автоматически применяются при старте сервера (`server/db.js → initTables()`). Если нужно применить миграцию отдельно (например, на staging-окружении без перезапуска):

```bash
mysql -u techrobot -p techrobot < server/migrations/001_blog.sql
```

Миграция идемпотентна — повторное применение безопасно.

### Безопасность блога

Контент пользовательских статей проходит **двойную санитизацию**:

1. **На сервере** при `POST`/`PUT /api/blog` через `sanitize-html` — удаляются все теги кроме whitelist: `p, h2, h3, h4, strong, em, u, a, ul, ol, li, img, code, pre, blockquote, br`. Атрибуты ограничены: `a:href` (http/https), `img:src` (только https), `alt`, `class`.
2. **На клиенте** при отображении (`blog-post.html`, превью в `blog-editor.html`, модалка в `admin.html`) через **DOMPurify** с тем же whitelist.

Rate limit на создание: **5 статей в час** на пользователя (по `userId`, не IP — корректно для NAT).
